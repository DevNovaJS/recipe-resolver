## ADDED Requirements

### Requirement: 요리 기록 저장
시스템은 요리 완료 시 기록을 저장해야 한다(SHALL). 기록은 메뉴(레시피), 사용 재료 목록(사용자 재료 ID + 사용량 + 단위), 요리 일시를 포함해야 한다(SHALL).

#### Scenario: 요리 기록 정상 저장
- **WHEN** 사용자가 레시피 ID, 사용 재료 목록(사용자 재료 ID + 사용량 + 단위), 요리 일시로 기록 저장 요청
- **THEN** 시스템은 요리 기록을 저장하고 생성된 기록 정보를 반환한다

#### Scenario: 존재하지 않는 레시피로 기록
- **WHEN** 사용자가 존재하지 않는 레시피 ID로 기록 저장 요청
- **THEN** 시스템은 404 에러를 반환한다

#### Scenario: 존재하지 않는 사용자 재료 ID
- **WHEN** 사용자가 존재하지 않는 사용자 재료 ID로 기록 저장 요청
- **THEN** 시스템은 404 에러를 반환한다

### Requirement: 재료 수량 자동 차감
시스템은 요리 기록 저장 시 사용자 재료의 수량을 자동으로 차감해야 한다(SHALL). 기록 저장과 수량 차감은 하나의 트랜잭션으로 처리해야 한다(SHALL). 차감은 요청 단위와 사용자 재료의 저장 단위가 동일한 경우에만 수행해야 한다(SHALL).

#### Scenario: 정상 차감
- **WHEN** 사용자가 토마토 2개를 사용한 요리 기록을 저장하고, 보유 토마토가 5개(단위 "개")인 경우
- **THEN** 시스템은 요리 기록을 저장하고 토마토 수량을 3개로 차감한다

#### Scenario: 보유 수량 부족 시 롤백
- **WHEN** 사용자가 토마토 5개를 사용한 요리 기록을 저장하고, 보유 토마토가 2개인 경우
- **THEN** 시스템은 트랜잭션을 롤백하고 400 에러와 함께 부족 재료 정보를 반환한다

#### Scenario: 단위 불일치 시 거부
- **WHEN** 사용자가 "g" 단위로 차감을 요청했으나 해당 사용자 재료의 저장 단위가 "개"인 경우
- **THEN** 시스템은 400 에러와 함께 단위 불일치 메시지를 반환하고 트랜잭션을 롤백한다

#### Scenario: 차감 후 수량 0
- **WHEN** 사용자가 보유 수량과 동일한 양을 사용한 경우
- **THEN** 시스템은 해당 사용자 재료의 수량을 0으로 설정한다 (삭제하지 않음)

### Requirement: 요리 기록 조회
시스템은 저장된 요리 기록을 조회할 수 있어야 한다(SHALL).

#### Scenario: 전체 기록 조회
- **WHEN** 사용자가 요리 기록 목록 조회 요청
- **THEN** 시스템은 요리 기록을 최신순으로 반환한다

#### Scenario: 기간별 기록 조회
- **WHEN** 사용자가 시작일과 종료일을 지정하여 기록 조회 요청
- **THEN** 시스템은 해당 기간의 요리 기록만 반환한다

### Requirement: 요리 통계 조회
시스템은 요리 통계를 조회할 수 있어야 한다(SHALL). 이번 달 요리 횟수, 자주 쓴 재료 TOP5, 자주 해먹은 메뉴 TOP5를 제공해야 한다(SHALL).

#### Scenario: 이번 달 통계 조회
- **WHEN** 사용자가 이번 달 통계 조회 요청
- **THEN** 시스템은 이번 달 요리 횟수, 자주 쓴 재료 TOP5, 자주 해먹은 메뉴 TOP5를 반환한다

#### Scenario: 기록 없는 월 통계 조회
- **WHEN** 사용자가 요리 기록이 없는 월의 통계 조회 요청
- **THEN** 시스템은 요리 횟수 0, 빈 TOP5 목록을 반환한다

### Requirement: 통계 배치 집계
시스템은 Spring Batch Job을 사용하여 요리 통계를 일 단위로 집계해야 한다(SHALL). 집계 결과는 별도의 통계 테이블에 저장해야 한다(SHALL).

#### Scenario: 배치 정상 실행
- **WHEN** 통계 집계 배치 Job이 실행된 경우
- **THEN** 시스템은 요리 횟수, 자주 쓴 재료 TOP5, 자주 해먹은 메뉴 TOP5를 집계하여 통계 테이블에 저장한다

#### Scenario: 배치 실패 후 재실행
- **WHEN** 통계 집계 배치가 실패한 후 재실행된 경우
- **THEN** 시스템은 멱등하게 동일 기간의 통계를 재집계한다 (중복 없음)

#### Scenario: 데이터 없는 기간 집계
- **WHEN** 집계 대상 기간에 요리 기록이 없는 경우
- **THEN** 시스템은 해당 기간의 통계를 0/빈 값으로 저장한다
